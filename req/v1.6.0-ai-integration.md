# AI Integration Requirements

Users should be able to integrate AI capabilities into their applications seamlessly. The following requirements outline the necessary features and functionalities for AI integration.

- User should be able to use Anthropic, OpenAi, Moonshot, Mistral.
- Users open Claude code, Codex, Kimi or Vibe (mistral) and add the Alipsa Accounting MCP
- It will be common that the MCP server will not be reachable for the AI CLI (since doing accounting is not a frequent task), hence care must be taken to not adversely affect the AI CLI when the accounting program is not running. 
- The MCP server contains tools for interaction, workflow descriptions and web resources to search 
- The MCP server is a remote MCP server. Change the current local MCP server functionality to remote MCP server functionality. We should not support local MCP servers anymore.
- The MCP server should provide users with the same functionality as the local MCP server, including the ability to run workflows, access tools, and search web resources. However, saving a voucher need to be confirmed by the user. get_company_info should be extended to include accounting_method. If a user has already created line items in a voucher but not yet saved it, the MCP server should be able to retrieve those line items and also know that they are not yet saved (and hence can be modified). This will allow the llm to provide more accurate and context-aware responses when assisting users with their accounting tasks.
- Add a guide for users on how to integrate AI capabilities into their applications, including step-by-step instructions and best practices to the user manual.
- The primary web resources for information during searches should be https://www.bokforingstips.se and https://www.skatteverket.se. The MCP server should prioritize these resources when providing information to users.
- The LLM should be provided with enough context of previous voucher registrations to provide accurate and relevant responses. This includes information about the user's accounting method, previously created line items, and any other relevant context that may impact the user's accounting tasks.
- some kind of "frosting glass" should be added to the MCP server interface to indicate that the user is interacting with an AI-powered system and prevent the user from modifying the system during an MCP operation. This will help users understand when they are receiving assistance from an AI. The frosting glass should be removed when the MCP operation is complete, allowing the user to resume normal interaction with the system.

Usage examples

Q: hur bokför jag ett inköp från amazon.de
A: Vid inköp av varor från ett annat EU-land än Sverige tillämpas reglerna om omvänd skattskyldighet för ett företag som lämnar sitt VAT-nummer till säljaren.

Omvänd skattskyldighet innebär att säljaren inte lägger på någon utgående moms på sin faktura till köparen och att det är köparen som i stället skall redovisa utgående moms och ingående moms på inköpsvärdet omräknat till redovisningsvalutan.

Eftersom du använder kontantmetoden gör du såhär:

| Konto | Benämning                                  | Debet   | Kredit  |
|-------|--------------------------------------------|---------|---------|
| 1931	  | Bankgiro, affärsbanken                     |         | 100 000 |
| 4010	  | Varuinköp	                                  | 100 000 |         |
| 4599	  | Justering, omvänd moms                     | 100 000 |         |
| 4515	  | Inköp varor EU, 25 %	                       |         | 100 000 |
| 2614  | Utgående moms omvänd skattskyldighet, 25 % |         | 25 000  |
| 2645	  | Ingående moms utland                       | 25 000  |         |

Se https://www.bokforingstips.se/artikel/bokforing/inkop-varor-eu.aspx
samt https://www.skatteverket.se/foretag/moms/kopavarorochtjanster/inkopfranandraeulander.4.361dc8c15312eff6fd1cfc6.html för mer information.

Q: jag har inte registrerat mitt VAT nummer how amazon.de.
A: Om fakturan innehåller **tysk moms (vanligen 19 %)** och varan skickades från Tyskland till ditt svenska momsregistrerade företag ska du normalt göra så här:

1. Bokför hela beloppet inklusive den tyska momsen som inköpskostnad.
2. Dra inte av den tyska momsen som svensk ingående moms.
3. Redovisa dessutom svensk förvärvsmoms på hela beloppet inklusive den tyska momsen.

Exempel: Amazonfaktura 100 euro + 19 euro tysk moms = 119 euro. Om det motsvarar 1 309 kr:

| Konto                                                |    Debet |   Kredit |
| ---------------------------------------------------- | -------: | -------: |
| 4056 Inköp varor 25 % EU*                            | 1 309,00 |          |
| 2645 Beräknad ingående moms på förvärv från utlandet |   327,25 |          |
| 2615 Utgående moms import av varor, 25 %             |          |   327,25 |
| 1930 Företagskonto                                   |          | 1 309,00 |

* Använd det kostnads- eller tillgångskonto som passar inköpet, men det måste hanteras så att underlaget kommer med i momsdeklarationens ruta 20. I din kontoplan används därför konto 4056 för detta.

I momsdeklarationen blir det:

* Ruta 20: 1 309 kr
* Ruta 30: 327 kr utgående moms
* Ruta 48: 327 kr ingående moms, om verksamheten har full avdragsrätt

Den svenska momsen tar alltså ut sig. Den tyska momsen blir däremot en faktisk kostnad. Skatteverket säger uttryckligen att när utländsk moms felaktigt tagits ut ska svensk moms ändå beräknas på priset **inklusive den utländska momsen**. [Skatteverket – köpa varor från andra EU-länder](https://www.skatteverket.se/foretag/moms/kopavarorochtjanster/inkopfranandraeulander/kopavarorfranandraeulander.4.3a7aab801183dd6bfd380005738.html)

Kontrollera dock fakturan noga: om momssatsen är **25 %** kan Amazon ha debiterat svensk moms genom sin distansförsäljning. Då är behandlingen en annan. Även vilket land varan faktiskt skickades från spelar roll – Amazon.de innebär inte automatiskt att det var ett inköp från Tyskland. Fakturans säljare, VAT-nummer, momssats och avsändningsland avgör.
