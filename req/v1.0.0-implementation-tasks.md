# v1.0.0 Implementation Tasks

Detta dokument bryter ned [roadmapen](/home/per/project/groovy-projects/accounting/req/v1.0.0-roadmap.md) till konkreta implementationstasks per fas.

## Arbetsprinciper

- Varje fas ska kunna avslutas i ett körbart tillstånd.
- Databasändringar ska alltid gå via migrationer och uppdatering av `schema_version`.
- Affärsregler ska testas i service-lagret före UI.
- Rapportdata ska byggas som återanvändbara DTO:er och inte direkt i mallarna.
- Bokföringskritiska ändringar ska loggas i `audit_log`.
- Testramverket för v1 är JUnit 6.0.3 kompletterat med `se.alipsa.groovy:groovier-junit:0.3.1`, och testkoden ska organiseras i `unit`, `integration` och `acceptance`.

## Tvärgående säkerhetsspår för H2

Detta spår ska löpa genom hela implementationen. Målet är inte att lova omöjlig manipulation av en lokal H2-fil, utan att göra manipulation svår, detekterbar och blockerande för bokföringskritiska flöden.

### Säkerhetstasks

1. Kör H2 endast i embedded mode och förbjud TCP-server, H2 Console och `AUTO_SERVER=TRUE`.
2. Lägg databasen i app-privat katalog med hårda filrättigheter.
3. Aktivera H2-kryptering och håll hemlighet/nyckel utanför databasens lagringskatalog.
4. Gör bokförda verifikationer append-only i service-lagret.
5. Implementera checksummor för bilagor, backup-manifest, importer och arkiverade rapporter.
6. Implementera hashkedja eller motsvarande integritetskontroll för bokförda verifikationer.
7. Implementera hashkedja eller daglig sammanfattningshash för `audit_log`.
8. Verifiera integriteten vid uppstart och före SIE-export, rapportarkivering och årsstängning.
9. Lägg till blockerande eller högt prioriterad varning när integritetskontroller fallerar.
10. Dokumentera uttryckligen att extern ändring av H2-filen är otillåten och att restore från backup är återställningsvägen.

### Definition of Done

- Appen exponerar inte H2 över nätverk eller lokal serverfunktion.
- Bokförda data kan inte ändras via normalt applikationsflöde.
- Appen kan upptäcka avvikelse mellan lagrad data och tidigare beräknade integritetsvärden.
- Integritetsfel stoppar eller varnar före bokföringskritiska operationer.
